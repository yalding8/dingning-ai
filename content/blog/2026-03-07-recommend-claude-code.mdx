我不是程序员出身。

这句话放在两年前说出来，大概没人会觉得有什么特别。但如果我告诉你，我在过去一年里独立构建了异乡人才（求职平台）、异乡点评（顾问评价系统）和你正在看的这个网站 dingning.ai，你可能会问：怎么做到的？

答案是 [Claude Code](https://claude.ai/download)。

## 什么是 Claude Code

Claude Code 是 Anthropic 推出的终端原生 AI 编程助手。注意关键词——终端原生。它不是一个带聊天框的网页，也不是 IDE 里的插件。你打开终端，输入指令，它就在你的项目目录里工作，直接读写文件、运行命令、理解整个代码库的上下文。

这种设计一开始让我有点不适应。毕竟大多数人对"AI 助手"的想象是一个对话窗口。但用了几天之后我就明白了：终端才是离代码最近的地方。Claude Code 不需要你复制粘贴代码片段给它看，它自己就在项目里，知道你的文件结构、依赖关系、配置细节。

## 我的实际使用场景

作为异乡好居的副总裁，我日常要处理大量业务事务，留给写代码的时间很有限。Claude Code 帮我做到的事情是：把想法快速变成可运行的产品。

拿异乡人才举个例子。这是一个面向海归求职者的平台，后端用 Python FastAPI，前端用 Next.js。整个推荐引擎——包括行为信号采集、Redis 缓存策略、贝叶斯评分降权逻辑——都是我在 Claude Code 里一步步构建出来的。我描述需求，它写代码，我审查，然后迭代。整个过程就像和一个非常懂技术的搭档结对编程。

异乡点评的情况更复杂一些。排序算法经历了五个大版本的迭代，从最初的简单加权到 Thompson Sampling、Wilson 置信下界，再到品类差异化权重配置。这些算法我能理解原理，但让我从零手写实现，老实说很困难。Claude Code 的上下文理解能力在这里发挥了巨大作用——它不仅能写出单个函数，还能理解整个评分系统的架构，确保新代码和现有模块无缝衔接。

## 为什么是 Vibe Coding

我把这种工作方式叫做 Vibe Coding。它不是"让 AI 替你写代码"那么简单。更准确的描述是：你负责产品判断和架构决策，AI 负责实现细节。你需要知道你要什么，需要能看懂代码在做什么，需要能判断方案的好坏。但你不需要记住每个 API 的参数顺序，不需要手动处理每一个边界情况。

这对非程序员背景的产品人来说意义重大。以前"一个人也可以成为一个公司"只是一句口号，现在它是可操作的现实。

## 几个实用细节

第一，Claude Code 对大型代码库的处理能力很强。异乡点评的后端加前端加上管理后台，文件数量早就过了几百个，它依然能准确定位到需要修改的位置。

第二，它和 Git 工作流配合得很好。我日常的开发节奏是：在 feature 分支上开发，Claude Code 帮我写代码和测试，然后提 PR、跑 CI、合并。整个流程和专业开发团队没什么区别。

第三，错误处理。代码跑不通的时候，把报错信息给它，它通常能直接定位问题并修复。这一点在调试 Docker 部署和数据库迁移的时候帮了我大忙。

## 适合谁

如果你是程序员，Claude Code 能显著提升你的开发效率。如果你像我一样不是程序员出身，但有明确的产品想法和基本的技术理解力，[Claude Code](https://claude.ai/download) 可能会彻底改变你的工作方式。

它不会把你变成程序员，但它能让你不再需要"先成为程序员"才能构建产品。

这是我目前最核心的生产力工具，没有之一。
