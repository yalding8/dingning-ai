## 为什么要建一个个人网站？

2025 年底，我开始密集使用 AI 辅助工作。写代码、做产品、带团队——AI 正在改变我工作的每一个环节。

这些实践和思考需要一个出口。

公众号是一个选择，但我想要更多的控制权——自定义设计、自定义域名、自定义功能。我想要一个**属于自己的数字根据地**。

于是，[dingning.ai](https://dingning.ai) 诞生了。

## 技术选型：我不懂，但 AI 懂

作为一个非程序员，我对前端框架的了解仅限于"听说过 React"。但这不妨碍我做出选择。

我的做法很简单：告诉 AI 我的需求，让它推荐技术栈。

最终的选型：

| 技术 | 用途 | 选择理由 |
|------|------|----------|
| Next.js 14 | 前端框架 | React 生态最成熟的全栈框架，SEO 友好 |
| TypeScript | 编程语言 | 类型安全，AI 生成的代码质量更高 |
| Tailwind CSS | 样式方案 | 原子化 CSS，不用写样式文件 |
| MDX | 内容管理 | Markdown + React 组件，写文章像写笔记 |
| Vercel | 部署平台 | Next.js 官方团队的平台，推送即部署 |

这些名词在建站之前我一个都不认识。但 Vibe Coding 的核心就是：**你不需要认识它们，你只需要知道它们能帮你解决什么问题。**

11 次提交。从脚手架初始化到完整上线，整个过程就是我和 AI 的一场对话。

### Phase 1：搭骨架

第一个大版本解决的是"有没有"的问题。

我告诉 AI：我要一个个人网站，包含首页、博客、项目、资源、关于我这五个页面。设计风格要简洁专业，配色用暖白底色、深灰文字、海军蓝强调色。

AI 给出了完整的页面结构：

- **首页**：个人介绍 + 精选文章 + Newsletter 订阅
- **博客**：文章列表，支持标签、日期、阅读时长
- **项目**：作品集展示
- **资源**：工具推荐
- **关于**：职业经历时间线

一次对话，五个页面骨架全部就位。

### Phase 2：写内容

骨架有了，但一个没有内容的网站等于没有。

第二个大版本引入了 MDX 博客系统。MDX 是一种特殊的 Markdown——在普通 Markdown 的基础上，可以嵌入 React 组件。对我来说，它的最大好处是：**写文章就像在记事本里打字，不需要碰任何代码。**

每篇文章是一个 `.mdx` 文件，开头写上标题、日期、标签，下面就是正文。AI 帮我写好了解析逻辑：读取文件、提取元数据、计算阅读时长、按日期排序。

同时接入了 Buttondown 做邮件订阅。读者在网站上填写邮箱，后台通过 API 自动添加到订阅列表。整个流程不到 50 行代码。

### 持续迭代：小步快跑

后续的每一次提交，都是针对具体问题的小迭代：

- 发现名字显示不一致 → 统一品牌名称
- 资源页推荐了没用过的工具 → 替换为真实在用的工具
- 产品推荐页太简略 → 为每个产品创建独立文章
- 定位不够清晰 → 重新定位为"国际教育 AI 布道者"，新增服务页面

每次迭代都很小，但方向很明确。这就是 Vibe Coding 的节奏——**不追求一次做完，而是每次做对一件事。**

## 非程序员建站的三个关键认知

### 1. 你的角色是产品经理，不是工程师

整个建站过程中，我没有写过一行代码。但我做了大量的**产品决策**：

- 首页放什么信息？精选文章怎么选？
- 导航栏用中文还是英文？
- 服务页面的培训模块怎么排列？
- 颜色方案用冷色调还是暖色调？

这些决策决定了网站的质感。AI 负责实现，我负责判断——这是一种全新的分工方式。

### 2. Git 是你的"后悔药"

Git 版本管理是我在建站过程中学到的最有价值的东西之一。

每一次修改都有记录。改坏了？回滚。想对比两个版本？diff。想看三天前的代码？checkout。

我的项目还配置了 Code Quality 门禁和 Branch Protection——这意味着每次代码变更都会经过自动检查，不符合规范的代码无法合入主分支。

这些"工程实践"听起来很专业，但本质上就是一句话：**让机器帮你把关，比靠人检查靠谱一百倍。**

### 3. 部署不是终点，是起点

很多人以为网站上线就结束了。但对我来说，上线才是开始。

截至目前，dingning.ai 已经有 17 篇文章，涵盖个人故事、技术实录、AI 实训复盘、工具推荐等多个方向。内容还在持续更新。

网站也在不断进化：从最初的单纯博客，到现在包含服务页面、项目展示、资源推荐的完整平台。每一次进化都是一次新的 Vibe Coding 实践。

## 写在最后

一年前，如果有人告诉我"你会搭一个技术博客"，我一定觉得这是开玩笑。

但今天，dingning.ai 就在那里运行着。17 篇文章，5 个功能页面，自动化的邮件订阅系统，Git 版本管理，代码质量门禁——一个正经的、生产级的网站。

它的每一行代码都是 AI 写的。但它的每一个决策都是我做的。

**Vibe Coding 不是让 AI 替你工作，而是让你用一种全新的方式工作。**

如果你也想试试，不需要学编程。你只需要：

1. 知道自己要什么
2. 能把需求描述清楚
3. 会判断结果对不对

剩下的，交给 AI。
