## 从想法到 MVP：如何用自然语言描述需求，让 AI 生成第一个可运行版本

我的起点，只是一个模糊的痛点：我身边许多优秀的留学生朋友，回国求职时总像在迷雾中摸索。海投简历石沉大海，信息渠道零散不全。我心想，能不能做一个平台，专门聚合适合留学生的优质岗位，并智能地推荐给他们？

**但我有个致命短板：我不是程序员。**

过去，这个想法会止步于脑海。但现在，我有了新的伙伴——AI。我的第一个任务，不是学习 Python 或 Django，而是学习如何清晰地“说话”。我打开对话窗口，开始描述我的构想：

“我需要一个网站，用户可以注册登录，填写自己的学校、专业、语言技能。后台需要有一个管理员，能发布工作岗位信息，岗位信息要包含公司、职位、要求、薪资。前台用户能看到一个岗位列表，并能根据一些条件简单筛选。”

这听起来像产品经理的需求文档，但对 AI 来说，这就是“原材料”。接着，我把它变得更具体、更可执行：

“请使用 Django 框架，创建一个项目。需要以下模型（Model）：
1. 用户模型（User），扩展 Django 自带的用户，增加学校、专业、语言分数等字段。
2. 岗位模型（Job），包含公司名、职位名称、职位描述、要求、薪资范围、工作地点等字段。
3. 请生成对应的数据库迁移文件。
4. 请为管理员创建一个后台管理界面，可以管理（增删改查）岗位信息。
5. 请创建一个简单的首页，列出所有岗位。”

几个小时后，在 AI 一步步的引导和代码输出下，我运行了 `python manage.py runserver`。浏览器里，一个简陋但功能完整的网站出现了。**我有登录注册，有后台管理，有数据列表。这就是我的 MVP（最小可行产品）。** 整个过程，我没有写一行代码，我只是在描述、追问和复述我的需求。这让我意识到，在 Vibe Coding 的世界里，**清晰的自然语言表达能力，就是最强大的编程语言。**

## 数据从哪里来：用 AI 编写爬虫，自动获取 10,645 个岗位数据

有了骨架，现在需要血肉——真实的岗位数据。手动收集不现实，我需要一个“机器人”去网上帮我抓取。这触及了我的知识盲区：网络爬虫。

我再次求助 AI。我的描述从“是什么”转向“怎么做”：
“我需要从几个目标招聘网站上获取岗位信息。请为我编写一个 Python 脚本。它需要能：
1. 模拟浏览器访问目标网址。
2. 解析网页的 HTML 结构，提取出公司名、职位名、薪资、要求等关键信息。
3. 将提取到的数据，以结构化的格式（比如 JSON）保存下来。
4. 注意设置访问间隔，避免给对方网站造成压力。”

AI 给了我一个基础脚本。但现实立刻给了我一击：网站结构比我描述的复杂，有反爬机制，数据格式也不统一。这是我遇到的第一个重大挑战。我不得不与 AI 展开“侦探式”的合作：

*   **我**：脚本运行出错了，它说找不到某个 HTML 标签。
*   **AI**：这可能是因为网站结构动态加载。我们可以尝试使用 `Selenium` 库来模拟真实浏览器操作，或者分析其网络请求，直接获取 JSON 数据。
*   **我**：（打开浏览器开发者工具，按照 AI 的指引，找到了数据接口）我找到了一个返回 JSON 的网址，里面的数据很规整！
*   **AI**：太好了，那我们可以直接请求这个接口，解析 JSON，这样更稳定高效。

这个过程反复了无数次。我学会了看网页结构（HTML），看网络请求，理解什么是 API、什么是 JSON。最终，我成功部署了多个爬虫脚本，它们像勤劳的矿工，日夜不停地从多个渠道采集数据。

**当数据库里的岗位数突破 10,645 个时，我感受到的不仅是数据的充实，更是一种“非程序员”也能驾驭复杂自动化任务的信心。** 工具（AI）解决了“如何实现”的问题，而我，作为产品的“大脑”，始终负责着“要什么”和“为什么”的战略决策。

## 推荐引擎的诞生：非程序员如何理解并实现 Thompson Sampling 算法

数据有了，下一个核心问题浮现：如何把最合适的岗位推荐给特定的学生？简单的关键词匹配太粗糙。我了解到，很多平台用“平均评分”排名，但这容易让新上架的优质岗位永无出头之日。

我需要一个更聪明的算法。AI 向我推荐了 **Thompson Sampling**，一种常用于解决“探索与利用”困境的算法。它既会推荐当前看起来最好的选项（利用已知最优），也会给新选项一定的曝光机会（探索未知可能）。

对于一个文科生来说，这听起来像天书。我没有直接让 AI 写代码，而是说：“**请用最简单的方式，给我讲一个故事，解释 Thompson Sampling 是什么，以及为什么它比简单的平均分排名更好。**”

AI 给我讲了一个“选择餐馆”的故事：
> 你有两家餐馆可选：A家去过10次，9次好评；B家是新开的，只去过1次，这次也是好评。如果只看平均分，A家90分，B家100分，你会永远选B吗？但B家可能只是运气好。Thompson Sampling 的做法是，它不会只看平均分，而是为每家餐馆估算一个“好评概率的分布区间”。A家的概率分布很集中（确信在90%左右），B家的分布很宽（可能在30%-100%之间）。每次选择时，它从这个“概率分布”里随机抽取一个值，然后选抽到值最高的那家。这样，B家虽然平均分高，但因其不确定性大，抽到极高值的概率也存在，因此它**有机会**被选中，但不会**永远**被选中。

这个故事瞬间点醒了我！这不正是我需要的吗？新岗位（B餐馆）需要被探索的机会，而稳定优质的岗位（A餐馆）也应得到应有的曝光。**理解“为什么”比理解数学公式更重要。**

理解了原理，实现就变成了“翻译”工作：
“根据刚才的 Thompson Sampling 原理，请用 Python 为我的岗位推荐系统实现这个算法。每个岗位都有‘展示次数’和‘被点击/申请次数’。请计算每个岗位的 Beta 分布参数，并根据抽样结果对岗位列表进行排序。”

就这样，一个包含着复杂概率思想的智能推荐引擎，在我这个非程序员的手中诞生了。它现在每天都在平台上默默工作，公平地为每一个岗位分配着流量。

## 测试是安全网：505 个单元测试如何保障系统稳定迭代

随着功能越来越多，我开始害怕。我怕改动了推荐算法，却无意中破坏了登录功能。这种恐惧在传统开发中，由经验丰富的工程师和严格的代码审查来规避。而我，只有 AI 和我自己。

AI 告诉我：“你需要编写单元测试。” 测试，就是给代码买的保险。我引入了 **Gate-Review 评审方法论**：任何新代码或修改，在并入主分支前，必须通过所有现有测试的检查（Gate），并由我（Review）进行最终的业务逻辑确认。

但怎么写测试？我继续描述：
“请为刚才的 Thompson Sampling 排序函数编写单元测试。测试应该覆盖：1. 当所有岗位都没有曝光时，是否随机排序；2. 当一个岗位点击率明显高于其他时，它是否最可能排在前面；3. 当一个新岗位加入时，它是否有机会出现在前列。”

AI 为我生成了测试代码的框架。我则负责扮演“破坏王”，不断思考各种极端情况：“如果分母为零怎么办？”“如果数据格式不对怎么办？”然后要求 AI 补充对应的测试。这个过程像在编织一张安全网。

**最终，这张网由 505 个单元测试构成。** 每当我增加一个新功能或修改一处逻辑，我只需运行一条命令，几分钟内，这505个“安全检查员”就会告诉我，我的改动是否安全。这让我敢于大刀阔斧地迭代和优化，因为我知道，一旦出错，测试会第一时间拉响警报。在 Vibe Coding 中，**测试不是可选项，而是让你能安心前行的必备安全网。**

## 部署与运维：从本地开发到生产环境的完整旅程

让网站跑在我的电脑上，和让全世界的人都能稳定访问，是两回事。我需要部署到生产环境。这又是一个全新的领域：服务器、Linux、Nginx、Docker、GitHub Actions……

我按照“分而治之”的策略，将这个大问题拆解：

1.  **选择环境**：我选择了一家云服务商，购买了一台虚拟服务器（VPS）。
2.  **配置服务器**：AI 给了我一份详细的清单和命令序列，教我如何通过 SSH 连接服务器，安装 Python、PostgreSQL、Redis、Nginx 等必要软件。
3.  **使用 Docker**：为了隔离环境、方便迁移，AI 建议我使用 Docker。我让它帮我编写了 `Dockerfile` 和 `docker-compose.yml` 文件，这就像一份标准化的“产品装箱单”。
4.  **自动化部署**：每次我在本地开发完，如何让服务器自动更新？我利用 GitHub Actions 设置了 CI/CD 流水线。我向 AI 描述：“当我向 Git 主分支推送代码时，请自动执行以下步骤：运行所有测试，通过后，在服务器上拉取最新代码，重新构建 Docker 容器并重启服务。”
5.  **监控与日志**：AI 教我如何查看服务器日志，如何设置简单的健康检查，确保服务在线。

当我在浏览器输入我的域名，看到平台顺利加载，并且爬虫正在后台安静地更新数据时，整个旅程完成了闭环。**从一个想法，到一个拥有上万数据、智能推荐、稳定运行的真实产品，我，一个非程序员，用 AI 作为主要工具，独立走完了全流程。**

## 写在最后

回顾这段从零到万的旅程，我深刻体会到，**AI 并没有让我“变成”程序员，而是让我成了一个“超级产品建造者”。** 我不再被“如何实现”的技术细节所阻挡，可以将绝大部分精力聚焦于产品构思、用户体验和业务逻辑这些更本质的问题上。

技术细节被“封装”在了我与 AI 的对话中。我的核心能力，演变为：
*   **精准定义问题的能力**
*   **系统性拆解任务的能力**
*   **理解并验证核心逻辑的能力**
*   **将想法转化为清晰、可执行的自然语言指令的能力**

异乡人才平台还在迭代。这段经历给我的最大启示是：**创造的壁垒，从未如此之低。** 下一个改变你所在领域的小工具、小平台，或许就可以由你，从今天与 AI 的一次对话开始。

*世界属于建设者，而如今，每个人都可以成为建设者。*
