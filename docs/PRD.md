# PRD: 自动化博客生成系统

## 1. 背景与目标

### 问题
Ning Ding 每天需要产出高质量博客内容来建立个人品牌（国际教育 AI 布道者），但手动写作耗时且难以保持日更节奏。

### 目标
构建一个自动化系统，每天从项目经验中自动选题、生成文章草稿、创建 PR 待人工审核后发布。

### 成功指标
- 每天自动生成 1 篇 1500-3000 字的博客草稿
- 文章基于真实项目经验，非凭空编造
- 人工审核通过率 ≥ 70%（即 10 篇中至少 7 篇可直接发布或微调后发布）
- 系统无人干预连续运行 ≥ 30 天

## 2. 用户与角色

| 角色 | 描述 | 操作 |
|------|------|------|
| 系统（GitHub Actions） | 每天定时触发 | 运行脚本、创建 PR |
| Ning Ding（审核者） | 博客作者 | 审核 PR、微调内容、合并发布 |
| 读者 | 博客访客 | 阅读文章 |

## 3. 功能需求

### P0（必须）
- [x] 从 `memories/` 读取项目 memory 作为写作素材
- [x] 扫描已发布文章，避免重复选题
- [x] 调用 Claude API 自动选题（返回 JSON）
- [x] 调用 Claude API 生成 MDX 格式文章
- [x] 自动生成 frontmatter（title, date, tags, excerpt）
- [x] GitHub Actions 每日定时触发
- [x] 生成后创建 PR 待审核
- [x] 支持手动触发 + 指定主题
- [x] `--dry-run` 模式仅选题不生成

### P1（应该）
- [ ] JSON 解析失败时的错误恢复（重试或 fallback）
- [ ] API 调用费用追踪 / 日志
- [ ] Memory 文件格式校验
- [ ] 生成文章质量自检（字数、格式、是否包含关键信息）

### P2（可以）
- [ ] 支持多语言生成（中英双版本）
- [ ] 自动配图（调用图片生成 API）
- [ ] 历史文章风格学习，保持文风一致性
- [ ] 文章自动发布到公众号

## 4. Out of Scope
- **不做** SEO 自动优化（当前阶段内容质量优先）
- **不做** 自动合并 PR（必须人工审核）
- **不做** 评论系统 / 读者互动分析
- **不做** 多作者支持（仅 Ning Ding 一人）
- **不做** 实时生成（仅定时批量）

## 5. 竞品分析

| 方案 | 优点 | 缺点 | 与本方案对比 |
|------|------|------|-------------|
| WordPress AI 插件 | 生态成熟 | 非 MDX，不够灵活 | 我们用 Next.js + MDX，版本化更好 |
| Jasper / Copy.ai | 专业写作 AI | SaaS 月费高，无法定制 prompt | 我们直接用 Claude API，成本更低且可控 |
| 手动 + ChatGPT | 灵活 | 需要人工操作，难以日更 | 我们全自动 + PR 审核流 |
| n8n / Zapier 自动化 | 可视化编排 | 额外依赖，黑盒 | 我们用 Python 脚本 + GitHub Actions，透明可控 |

## 6. 数据流

```
memories/*.md → 脚本读取 → Claude API 选题 → Claude API 生成 → MDX 文件 → Git commit → PR 创建 → 人工审核 → 合并发布
```

## 7. 风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| Claude API 不可用 | 当天无文章 | GitHub Actions 失败通知，次日重试 |
| 生成内容与事实不符 | 品牌信誉 | 人工 PR 审核必须通过 |
| API 费用超预算 | 成本 | 配置 max_tokens 上限，监控用量 |
| Memory 素材耗尽 | 选题重复 | 定期补充新项目 memory |
